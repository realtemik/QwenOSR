# QwenOSR

MVP-десктопное приложение для Windows: записывает системный звук, но **не отправляет его автоматически**. Вы сами нажимаете кнопку **«Отправить запись в Qwen»**, когда нужно проанализировать накопленный звук. Можно записывать звук в zoom, яндекс телеметре, т.д. и отправлять текст в ии.

## Как работает

```text
системный звук → накопление записи → кнопка «Отправить запись» → Whisper → текст → Ollama/qwen2.5 → ответ в чате
```

Qwen2.5 в Ollama обычно принимает текст, а не сырой звук. Поэтому сначала используется Whisper/faster-whisper для расшифровки аудио.

## Требования

- Windows 10/11
- Python 3.10 или 3.11
- Установленный Ollama https://ollama.com/download/OllamaSetup.exe
- Загруженная модель Qwen2.5:3b (можно и другую, но надо ее скачать и указать в config):

```bash
ollama pull qwen2.5:3b
ollama serve
```

## Установка (актуально только для MSYS2 MINGW64)

```bash
cd system_audio_qwen_chat
python -m venv .venv
source ./.venv/Scripts/activate
pip install -r requirements.txt
```

## Как пользоваться

1. Запустите Ollama:

```bash
ollama serve
```

2. Запустите приложение:

```bash
python ./app/main.py
```

3. Нажмите **«Начать запись»**.
4. Включите видео, звонок, лекцию или другой системный звук.
5. Когда захотите отправить накопленную запись на анализ, нажмите **«Отправить запись в Qwen»**.
6. Приложение:
   - возьмет накопленное аудио;
   - очистит буфер записи;
   - распознает речь через Whisper;
   - отправит текст в qwen2.5;
   - выведет ответ в чат.
7. После отправки запись продолжит накапливаться заново.

## Кнопки

- **Начать запись/стоп** — начинает слушать системный звук и копить аудио, останавливает всю запись.
- **Отправить запись в Qwen** — вручную отправляет накопленную запись в Whisper, затем текст в Qwen.
- **Очистить запись** — удаляет накопленное аудио без отправки.
- **Очистить чат** — очищает только окно чата.
- **Отправить сообщение** — отправить супер мега уточняющее сообщение

## Настройка

Файл `config.json`

```json
{
  "ollama_base_url": "http://localhost:11434",
  "ollama_model": "qwen2.5:3b",
  "whisper_model": "base",
  "language": "ru",
  "chunk_seconds": 1,
  "sample_rate": 16000,
  "min_text_length": 8,
  "min_record_seconds": 1.0,
  "system_prompt": "промт"
}
```

## Возможные проблемы

### Ollama не отвечает

Проверьте:

```bash
ollama serve
curl http://localhost:11434/api/tags
```

### Модель Qwen не найдена

```bash
ollama pull qwen2.5:3b
```

### Нет системного звука

Проверьте, что звук действительно идет через текущее устройство вывода Windows. Приложение использует loopback-захват системного динамика.

## Ограничения MVP

- Нет выбора аудиоустройства в интерфейсе.
- Приложение не различает говорящих/музыку. Включено должно быть только то, что надо. Ваш микрофон не записывается.
- Во время обработки накопленной записи новая запись временно не пишется.
- Qwen анализирует только текст после Whisper, не аудио напрямую.
